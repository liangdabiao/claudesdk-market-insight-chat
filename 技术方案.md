# MarketInsightChat — 技术方案

## 项目概述

基于 **Claude Agent SDK** 的用户需求洞察与产品机会分析平台。用户输入产品/服务描述后，系统调用 `insight` 技能，执行三段式洞察框架（用户画像 → 情绪洞察 → 产品机会），10分钟内输出传统方法需数周才能完成的深度分析报告。

## 架构

```
用户浏览器 ←WebSocket→ Express Server ←→ Claude Agent SDK ←→ Insight Skill
                          ↓
                   insight_reports/  (结构化洞察报告)
```

- **后端**: Express + WebSocket + Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`)
- **前端**: React + Tailwind CSS + Vite，三栏布局（洞察报告侧栏 + 对话区 + 执行日志）
- **AI 引擎**: 通过 Skills 系统驱动 `insight` 技能，执行三段式洞察框架

## Insight 技能核心：三段式洞察框架

```
┌──────────────────┐    ┌──────────────────┐    ┌──────────────────┐
│ 阶段一：用户画像    │ →  │ 阶段二：情绪洞察    │ →  │ 阶段三：产品机会    │
│                  │    │                  │    │                  │
│ 4类潜在用户       │    │ 痛点/痒点/爽点     │    │ P0/P1/P2优先级    │
│ +核心用户判断     │    │ +可直接用的文案     │    │ +可落地的清单      │
└──────────────────┘    └──────────────────┘    └──────────────────┘
```

| 阶段 | 核心目标 | 输出成果 |
|------|----------|----------|
| **用户画像识别** | 找到最有价值的目标用户，避开"抓住所有人"陷阱 | 4类具象用户画像 + 核心用户推荐（复购频次、抗周期性、成瘾性） |
| **情绪动因分析** | 从痛点/痒点/爽点三维度拆解真实动机 | 可直接用于营销文案的情绪洞察表达 |
| **产品机会转化** | 将情绪洞察转化为优先级排序的可落地需求 | P0/P1/P2级产品机会清单（含成本/难度/预期效果） |

## 产出物结构

```
insight_reports/[产品名称]/
├── README.md                  # 概览导航
├── executive_summary.md       # 执行摘要（核心发现 + Top 3 机会）
├── user_personas.md           # 用户画像分析
├── emotional_insights.md      # 情绪动因分析（痛点/痒点/爽点）
├── product_opportunities.md   # 产品机会清单（P0/P1/P2）
└── full_report.md             # 完整洞察报告
```

## Skills 部署

```
.claude/skills/
└── insight/
    ├── SKILL.md                          # 技能定义（三段式框架 + 工作流程 + 完整案例）
    ├── references/
    │   ├── prompt_templates.md           # 提示词模板（用户画像/情绪洞察/产品机会 + 行业适配）
    │   └── quick_reference.md            # 快速速查表
    └── assets/
        └── sample_report_template.md     # 报告模板
```

**特点**: insight 是纯文本技能，无外部 API 依赖，完全依赖 Claude 语言能力 + 内置提示词框架。不需要 Bash/WebSearch 等工具。

## 技术栈

| 组件 | 选型 |
|------|------|
| AI Agent | `@anthropic-ai/claude-agent-sdk` v0.1.11 |
| HTTP/WS 服务 | Express + ws |
| 前端框架 | React 18 + TypeScript |
| 样式 | Tailwind CSS 4 |
| 构建 | Vite 6 |
| 文件上传 | multer（支持 .pdf/.doc/.docx/.txt/.md/.csv） |
| Markdown 渲染 | react-markdown |

## Agent 配置

```typescript
// server/agent-client.ts
allowedTools: ["Skill", "Read", "Write", "Glob", "Grep", "TodoWrite"]

settingSources: ["project"]  // 自动加载 .claude/skills/insight
```

**说明**: insight 技能不需要 Bash（无外部脚本调用）、不需要 WebSearch（不联网搜索）。工具集精简为：Skill（调用洞察技能）、Read/Write（读写报告文件）、Glob/Grep（查找参考资料）、TodoWrite（任务跟踪）。

## System Prompt 要点

```
你是 MarketInsightChat — 专业的用户需求洞察与产品机会分析 AI 助手。

你拥有 insight 技能，能执行三段式用户洞察框架：
1. 用户画像识别 — 生成4类具象用户画像，标注核心用户
2. 情绪动因分析 — 痛点/痒点/爽点三维分析，输出可用营销文案
3. 产品机会转化 — P0/P1/P2优先级排序的可落地产品方向

工作流程：
- 收集用户的产品/服务描述
- 按三段式框架逐步执行分析
- 最终输出结构化洞察报告到 insight_reports/ 目录

行业适配：餐饮食品、SaaS/B2B、电商消费品、内容教育等
```

## 端口与 API

| 端点 | 说明 |
|------|------|
| `GET /api/health` | 健康检查 |
| `GET /api/reports` | 获取洞察报告文件树 |
| `POST /api/upload` | 上传参考资料 |
| `WS /ws` | WebSocket 实时通信 |
| `GET /insight_reports/*` | 静态文件服务（报告） |

- 端口：**3008**
- 上传目录：`uploads/`
- 报告目录：`insight_reports/`
